Section 1: User Flow Design

Authentication and checkout flow for package selection

Flow A: Package Selection (Unauthenticated User)

User clicks "Get Started" on any package
1 User on /pricing clicks "Get Started" on Premium ($499)
2 Redirect → /auth/signup?plan=premium
3a User signs up (name, email, password only)
3b OR clicks "Already have account? Log in" → /auth/signin?plan=premium
4 After successful auth → Create Stripe checkout session
5 Immediate redirect → Stripe Hosted Checkout
6 Payment success → /payment/success → Dashboard

Flow B: Direct Signup (No Package Selected)

User clicks generic "Get Started" or goes directly to signup
1 User on landing page clicks "Get Started" (no plan param)
2 Redirect → /pricing (see packages first)
3 User selects package → Flow A continues

Flow C: Authenticated User Selects Package

Already logged-in user clicks package
1 User on /pricing clicks "Get Started" on Complete ($999)
2 Create Stripe checkout session immediately
3 Redirect → Stripe Hosted Checkout
4 Payment success → /payment/success → Dashboard
